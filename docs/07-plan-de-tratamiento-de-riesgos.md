# 07 — Plan de Tratamiento de Riesgos

Las acciones propuestas se basan en los controles definidos por ISO/IEC 27002:2022, ISO/IEC 27701, RGPD y MicroPILAR.

| Riesgo | Control aplicado | Acción recomendada | Responsable | Prioridad |
|--------|------------------|--------------------|-------------|-----------|
| Acceso no autorizado | 5.15 / 8.2 Gestión de privilegios | Implementar autenticación multifactor (MFA) | TI Hospital | Alta |
| Interrupción del servicio | 5.29 / 8.14 Continuidad del servicio | Redundancia del servidor Open-RMF y Wi-Fi | Infraestructura TI | Alta |
| Vulnerabilidades técnicas | 8.5 Parcheo | Plan periódico de actualización de firmware y RMF | DevOps / Proveedor | Alta |
| Robo de información | 8.22 / Criptografía | Cifrado de datos en tránsito y reposo (TLS + AES256) | Seguridad / Redes | Alta |
| Riesgos de privacidad | ISO 27701 + RGPD | Registro DPIA y control de logs sensibles | DPO + TI | Media |
